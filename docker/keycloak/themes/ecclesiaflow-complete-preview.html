User flow

Authentication journey

Two distinct experiences based on role. Admin sees the full brand pitch; users get a simple, welcoming interface.

Admin flow

1. Admin registers church
2. Configures services & groups
3. Accepts member requests
4. Manages private church data

User flow

1. User creates account
2. Confirms email + sets password
3. Browses churches & public programs
4. Requests membership to a church
5. Admin accepts → full access
